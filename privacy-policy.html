Политика конфиденциальности · Редакция от 19.07.2026

Полный текст также доступен внутри приложения на русском, казахском, английском и китайском языках (Профиль → Настройки → Политика конфиденциальности).

1. Какие данные мы обрабатываем

  • Имя, номер авто и VIN — вводятся вами вручную в профиле.
  • Доверенные контакты (имя и номер телефона) — для отправки SOS-SMS.
  • Геолокация — включается только при записи видео или отправке SOS; координаты добавляются к метаданным записи и SMS.
  • Аудио и видео — сохраняются локально на вашем устройстве; никуда не передаются.
  • История остановок и статистика — хранятся локально в зашифрованном хранилище.
  • Email, имя и фото профиля Google — только если вы вошли в аккаунт (см. раздел 5).

2. Как мы храним данные

Большинство данных (профиль, контакты, история, видео) хранится исключительно на вашем устройстве с использованием Android Keystore и Encrypted DataStore и никуда не передаётся. Отдельные технические данные обрабатываются на нашем сервере — см. раздел 5.

Видеозаписи защищены SHA-256 хэшом и цифровой подписью (EC P-256), что гарантирует целостность файла.

3. Доступ к данным

Приложение запрашивает следующие разрешения:

  • CAMERA, RECORD_AUDIO — для видео/аудиозаписи.
  • ACCESS_FINE_LOCATION — для добавления координат к SOS-сообщению.
  • CALL_PHONE — для набора номеров 102 и 112.
  • READ_MEDIA_VIDEO — для отображения записей из галереи (Android 13+).

Отправка SOS открывает штатное SMS-приложение с уже заполненными получателями и текстом — само приложение SMS не отправляет и не запрашивает для этого отдельное разрешение.

Каждое разрешение запрашивается в момент использования функции, а не при установке.

4. Соответствие Закону РК №94-V

  • Биометрические данные (отпечаток пальца, Face ID) не хранятся — используется только системный BiometricPrompt.
  • Вы вправе в любой момент удалить все свои локальные данные через приложение (Настройки → Удалить все данные).
  • Если вы вошли в аккаунт, вы также вправе полностью удалить сам аккаунт (Профиль → Удалить аккаунт) — это необратимо удаляет ваш email/Google-профиль с серверов Firebase вместе со всеми локальными данными.
  • Сбор данных осуществляется только с вашего явного согласия (через запросы разрешений ОС).

5. Сторонние сервисы

  • Google Play Billing — для обработки подписок. Google может получать данные транзакции согласно своей политике конфиденциальности.
  • Firebase Authentication — при входе в аккаунт (Google или email/пароль) обрабатывает email, имя и уникальный идентификатор пользователя на серверах Google/Firebase.
  • Firebase Crashlytics — включается только по вашему согласию в настройках; при включении собирает отчёты о сбоях приложения.
  • mybuh.kz — один сетевой запрос для получения актуального значения МРП (не чаще 1 раза в 30 дней). Личные данные не передаются.
  • RoadShield_backend (наш собственный сервер, api.roadshield.kz) — при каждом запуске отправляется токен проверки целостности устройства (Play Integrity); при оформлении подписки — токен покупки и идентификатор пользователя Firebase для независимой проверки статуса Premium. Срок действия Premium-подписки хранится на сервере, привязанный к вашему Firebase UID.

Удаление данных и аккаунта

Удалить локальные данные (профиль, контакты, историю, статистику) можно в любой момент внутри приложения: Настройки → Удалить все данные.

Если вы зарегистрированы в приложении (email или Google), вы можете полностью удалить аккаунт и все связанные с ним данные: Профиль → Удалить аккаунт. Действие необратимо.
